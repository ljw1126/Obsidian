
안녕하세요, 류성현입니다.  
ATDD in Legacy Code 2기 개설 소식을 전해드립니다.

이번 과정은 **레거시 환경에서 신뢰할 수 있는 인수 테스트를 구축**하는 데 중점을 두고 설계되었습니다.

그동안 강의를 운영하며 단순히 클린 코드 습관을 익히는 것을 넘어, 많은 분들이 겪는 **‘테스트 부재로 인한 배포 불안’** 문제를 실질적으로 해결할 과정이 필요하다고 생각해왔습니다.  
이 과정에서는 **ATDD와 인수 테스트를 레거시 코드에 적용하는 방법을 배우고, 매주 주어지는 미션과 실전 워크숍을 통해 직접 부딪히며 실무 감각을 익히는 것**을 목표로 합니다.  
  
```text
[기존 ATDD 강의와의 차별점]
이번 신규 과정은 기존 ‘ATDD, Clean Code with Spring’ 과정이 대체되거나 변경된 것이 아니며, 두 과정은 서로 다른 초점을 가진 독립된 과정입니다.  

  - 1) ATDD, Clean Code with Spring: 스프링 프로젝트에서 인수 테스트 기반으로 객체지향·클린 코드를 실천하며, 좋은 코드 작성 습관 강화에 초점을 둡니다.  
  - 2) ATDD in Legacy Code: 복잡한 레거시 코드 환경에서 인수 테스트를 구축하고, 테스트 기반으로 안정적이고 확장 가능한 개발·배포 환경을 만드는 경험에 집중합니다.
```
